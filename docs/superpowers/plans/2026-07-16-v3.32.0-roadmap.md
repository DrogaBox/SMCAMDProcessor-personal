# v3.32.0 Roadmap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v3.32.0 with temperature offset profiles, dead code removal, chart height centralization, CHANGELOG update, and System Info expansion.

**Architecture:** 6 independent tasks across kext (AMDRyzenCPUPowerManagement) and app (AMD Power Gadget). Each task can be built and tested independently. Order: kext first (profile changes), then app cleanup, then app features, then documentation.

**Tech Stack:** XNU kext C++ (AMDRyzenCPUPowerManagement), SwiftUI (AMD Power Gadget app)

**Interfaces:**


- [ ] **Step 2: Simplify MenuCoordinator.setHeight to read from UserDefaults**

Replace the hardcoded switch in `MenuCoordinator.setHeight`:
```swift
private func setHeight(_ heightType: String) {
    let key = "chart_h_" + chartId
    // Heights are set via ResizableChart's stored values.
    // The coordinator only writes and posts notification.
    // Height values come from the chart's stored defaults.
    NotificationCenter.default.post(name: NSNotification.Name("DashboardLayoutChanged"), object: nil)
}
```

(Note: this works because `ResizableChart.init()` already reads the UserDefaults key and falls back to `medium` if no saved value exists. The coordinator just needs to post the notification.)
